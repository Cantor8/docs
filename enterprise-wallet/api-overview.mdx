The **C8 Enterprise Wallet API** is a REST interface to all wallet operations exposed by the service. Browse the endpoints in the left sidebar — each page includes an interactive playground where you can send requests directly from the docs.

## Base URL and auth

<CardGroup cols={2}>
  <Card title="Base URL" icon="link">
    Your deployment's host — e.g. `https://wallet-api.your-domain.com`.
  </Card>

  <Card title="Authentication" icon="shield-halved">
    `Authorization: Bearer <JWT>` when `AUTH_TYPE` is set. See [Authentication](/enterprise-wallet/authentication).
  </Card>
</CardGroup>

## Conventions

<AccordionGroup>
  <Accordion title="Authentication" icon="key">
    All protected endpoints expect a Bearer JWT in the `Authorization` header. In `noop` mode the header is ignored.
  </Accordion>

  <Accordion title="Response shape" icon="brackets-curly">
    Success responses return JSON bodies described by the schema on each endpoint page. Errors return `BadRequest`, `Unauthorized`, `Forbidden`, `NotFound`, `Conflict`, `UnprocessableEntity`, or `InternalServerError` envelopes with `error` and `timestamp` fields.
  </Accordion>

  <Accordion title="Idempotency" icon="rotate">
    Submitting the same `commandId` for a transfer will return the existing command status rather than create a duplicate.
  </Accordion>

  <Accordion title="Pagination" icon="list">
    History and transfer endpoints accept `startDate`, `limit`, and `direction` query parameters.
  </Accordion>
</AccordionGroup>

<Tip>
  Each endpoint page in the sidebar includes an interactive **Try it** playground. Set your base URL and Bearer token once — the playground will reuse them across requests.
</Tip>
