## Prepare dependencies

<Warning>
  While local `sbt` works automatically, **Docker** builds run in an isolated environment and cannot access your local `~/.m2` cache. You must manually copy the built SDK artifacts into the project's `libs/` folder so the Dockerfile can see them.
</Warning>

Run the following commands from the project **root**:

```bash
# 1. Create the local library injection folder
mkdir -p libs

# 2. Copy the 'tech' group from your local Maven cache
cp -r ~/.m2/repository/tech libs/

# 3. Verify the structure (optional)
# You should see the wallet-sdk-jvm artifacts inside
ls -R libs/tech/cantor8/cantonwallet/wallet-sdk-jvm/
```

<Info>
  **Why is this necessary?** The `Dockerfile` is configured to copy the contents of `libs/` into the container's internal Maven repository. Without this step, the Docker build fails with `Unresolved Dependency` errors.
</Info>

## Build and run

Docker configurations live in the `docker/` directory, but commands should be run from the project root.

### Verify the build

Run a clean build of the Docker image:

```bash
docker/verify-build.sh
```

### Run with Docker Compose

Spin up the service locally:

```bash
docker compose -f docker/docker-compose.yml up -d
```
