DAML packages are built with **`dpm`**, a DAML package manager wrapper around the DAML SDK.

## Build

Build a single package:

```bash
cd contracts/c8/tokens/ceth-rules
dpm build
```

## Run DAML tests

```bash
cd contracts/token-test
dpm test
```

## Deploy

<CodeGroup>

```bash DevNet
./scripts/deploy-devnet.sh all
```

```bash MainNet
./scripts/deploy-mainnet.sh all
```

</CodeGroup>

After deploying new DARs, **recreate factories** so the registry and clients use the current rules package:

```bash
./c8t create-factory
```

## End-to-end testing

The `e2e/` environment uses Docker Compose and isolates signing keys per party:

<CardGroup cols={2}>
  <Card title="registry" icon="server">

  </Card>

  <Card title="admin" icon="user-shield">
    Admin key only.
  </Card>

  <Card title="party1" icon="user">
    party1 key only.
  </Card>

  <Card title="party2" icon="user">
    party2 key only.
  </Card>
</CardGroup>

Run the full suite:

```bash
cd e2e
./setup.sh
./run-all.sh
./teardown.sh
```

Run selected tests:

```bash
./run-all.sh 05 09 14
./run-all.sh --from 09
```

The suite covers split, merge, transfers, freeze behavior, admin operations, direct transfers, allocation, swap flows, rebateCC mint / redeem, and failure cases.
