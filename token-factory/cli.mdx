The repository ships a root `c8t` wrapper around `scripts/c8t_ops.py` for everyday operations.

## Selecting an environment

Set `C8T_ENV` before each command:

```bash
C8T_ENV=devnet  ./c8t holdings
C8T_ENV=mainnet ./c8t holdings
```

## Command reference

<CodeGroup>

```bash Setup
./c8t create-factory
./c8t create-token-config
./c8t parties
```

```bash Token lifecycle
./c8t mint <party> <amount>
./c8t burn <holding_cid>
./c8t burn-all <party>
./c8t admin-burn <holding_cid> <reason>
./c8t migrate <holding_cid>
```

```bash Holdings
./c8t holdings
./c8t balance <party>
./c8t split <owner> <holding_cid> <amount_1> <amount_2>
./c8t merge <owner> <amount> <holding_cid_1> <holding_cid_2>
```

```bash Transfers
./c8t transfer <sender> <receiver> <amount>
./c8t accept <receiver> <transfer_instruction_cid>
./c8t reject <receiver> <transfer_instruction_cid>
./c8t withdraw <sender> <transfer_instruction_cid>
./c8t instructions
```

```bash AutoAccept
./c8t create-auto-accept <receiver>
./c8t cancel-auto-accept <party_or_auto_accept_cid>
./c8t auto-accepts
```

```bash Freeze
./c8t token-config
./c8t freeze
./c8t unfreeze
./c8t freeze-party <party>
./c8t unfreeze-party <party>
```

```bash Allocation & settlement
./c8t allocate <sender> <receiver> <amount> [executor]
./c8t settlement-accept <receiver> <proposal_cid>
./c8t settlement-reject <receiver> <proposal_cid>
./c8t settlement-execute <settlement_cid> [executor]
./c8t settlement-cancel <settlement_cid> [executor]
./c8t settlement-withdraw <sender> <proposal_cid>
./c8t allocation-withdraw <sender> <allocation_cid>
```

</CodeGroup>

## Party aliases

Party aliases can be loaded from the `scripts/parties/` YAML files.

<Warning>
  In production, never publish party signing keys or environment files.
</Warning>
