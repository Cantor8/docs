---
title: "Production checklist"
description: "What to verify before going live with a token deployment."
---

Run through this list before promoting Token Factory to a production environment.

## Deployment

<AccordionGroup>
  <Accordion title="DARs deployed" icon="box-open">
    Deploy holding and rules DARs for the target token.
  </Accordion>

  <Accordion title="Factory contracts in place" icon="industry">
    Create or verify factory contracts for each token.
  </Accordion>

  <Accordion title="TokenConfig created" icon="sliders">
    Create `TokenConfig` for tokens that respect global / per-party freeze.
  </Accordion>

  <Accordion title="Registry config updated" icon="server">
    Configure registry token JSON with current package / template IDs and factory contract IDs.
  </Accordion>
</AccordionGroup>

## Network and access

<AccordionGroup>
  <Accordion title="Private admin API isolated" icon="lock">
    Keep the private admin API on an internal network or behind a protected route.
  </Accordion>

  <Accordion title="M2M JWT enabled" icon="key">
    Enable M2M JWT authentication for private admin operations.
  </Accordion>

  <Accordion title="Public clients restricted" icon="globe">
    Verify wallet clients use only public CIP-0056 endpoints.
  </Accordion>
</AccordionGroup>

## Quality and operations

<AccordionGroup>
  <Accordion title="Tests passing" icon="flask">
    Run DAML unit tests and e2e tests against the target environment.
  </Accordion>

  <Accordion title="Signer isolation confirmed" icon="shield">
    Confirm signer isolation for admin, party, and validator workflows.
  </Accordion>

  <Accordion title="No secret leakage" icon="file-shield">
    Avoid exposing `.env.*.secret`, party signing keys, contract blobs, and environment-specific private files.
  </Accordion>
</AccordionGroup>

<Check>
  Once all items above are green, the deployment is ready for production traffic.
</Check>
